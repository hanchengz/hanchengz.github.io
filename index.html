Vue实例

每个Vue应用都是通过Vue函数创建一个新的Vue实例开始的。

数据与方法

在创建一个新的Vue实例时，可以传入一个选项对象，来创建我们想要的行为。在创建时它向Vue的响应式系统中加入了data对象中的所有属性，当其中的属性的值发生变化时，视图也会实时的响应，更新为新的状态。
值得注意的是，当数据发生改变时，视图会重新渲染，但是仅有实例被创建时data中存在的属性是响应式的。这意味着如果我们在实例化一个Vue对象后再添加一个新的属性，那么这个新增的属性发生改变时不会触发视图的更新。
除了data属性，Vue实例还暴露了一些有用实例属性与方法。它们都有前缀$，以便与用户定义的属性区分开来。

实例生命周期

每个Vue实例在创建之前都要经过一系列的初始化过程。例如需要设置数据监听，编译模版，挂载实例到dom，在数据变化时更新dom等。同时也会运行一些叫做生命周期钩子的函数。
例如created钩子可以用来在一个实例被创建后执行代码。

还有一些其他的钩子，在实例生命周期不同的场景下调用。如mounted、updated、destroyed。钩子的this指向调用它的Vue实例

不要在选项属性或回调上使用箭头函数，比如created: () => console.log(this.msg),或vm.$watch(‘a’,newValue => this.myMethod())。因为箭头函数是和父级上下文绑定在一起的，this不会是预想中的值。

模版语法

插值

数据绑定最常见的形式就是使用“Mustache”语法（双大括号）的文本插值。

Mustache标签将会被替换为相应数据对象上msg属性的值。当msg的值发生改变时，插值处的的内容都会更新。
也可以通过使用v-once指令来执行一次性的插值，这样一来数据发生改变时，插值处的内容也不会更新，但这会影响到这个节点的所有数据绑定。

原始HTML

双大括号会将数据解释为普通文本，而非HTML代码，如果需要输出原始的HTML，则要使用v-html指令。

这样一来这个div的内容将会被替换为属性值rawHtml，要注意的是直接作为HTMl，会忽略解析属性值中的数据绑定。同时不能使用v-html来复合局部模版。

特性

Mustache语法不能作用在HTML特性上，需要使用v-bind指令来完成。

使用javascript表达式

对于所有的数据绑定，Vue.js都提供了完全的javascript表达式支持。

模版表达式都放在沙盒中，只能访问全局变量的一个白名单，比如Math和Date。不能在模版表达式中试图访问用户定义的全局变量。

指令

指令是带有v-前缀的特殊属性。指令属性的值预期是单个JavaScript表达式（不包括v-for）。指令的作用是当表达式的值发生改变时，将其产生的连带影响，响应式地作用于DOM。

参数

一些指令可以接收一个参数，在指令名称之后以冒号表示。

在这里href是参数，使用v-bind指令将该元素的href属性与表达式url的值绑定在一起。
v-on用于监听DOM事件，参数为事件名。

修饰符

用于指出一个指令应该以特殊方式绑定。如 .prevent修饰符告诉v-on指令对于触发的事件调用event.preventDefault()

关于对象

看了一下创建对象的几种方式。

一、工厂模式

虽然这个函数可以根据传入的参数来构造多个相似的对象，但是没有解决对象识别问题(即判别该对象是什么类型)。

二、构造函数模式

person1和person2分别保存着Person的一个不同的实例。这两个对象都有一个constructor(构造函数)属性，该属性指向Person。

该模式与工厂模式的不同在于：

  • 没有显式的创建对象
  • 直接将属性和方法赋值给了this对象
  • 没有return

相比于工厂模式的无法识别对象类型，该模式可以使用instanceof操作符检测到对象类型。
创建自定义的构造函数意味着可以将它的实例标识为一种特定的类型。

使用构造函数模式必须使用new操作符，以这种方式调用构造函数会经历以下四个过程：

  1. 创建一个新的对象
  2. 将构造函数的作用域复制给新对象（因此this对象就只指向了这个新对象）
  3. 执行构造函数中的代码（为这个新对象添加属性）
  4. 返回新对象

在使用构造函数模式的时候需要注意到，每个方法都要在每个实例上重新创建一次，上面的person1和person2中都有一个sayName方法，但这两个方法并不是同一个function的实例。在ECMAScript中，函数也是对象，因此每定义一个函数，也就是实例化了一个对象。

三、原型模式

我们创建的每个函数都有一个prototype(原型)属性，这个属性是一个指针，指向一个对象，该对象包含可以由特定类型的所有实例共享的属性和方法。prototrpe就是通过调用构造函数而创建的那个对象实例的原型对象。使用原型对象的好处是可以让所有的对象实例共享它所包含的属性和方法。

所有的原型对象都会自动获得一个constructor(构造函数)属性，这个属性是一个指向prototype属性所在函数的指针。Person.prototype.constructor指向Person。

当调用构造函数创建一个新的实例后，该实例的内部会包含一个指针（内部属性），指向构造函数的原型对象。ES5中该指针为[[Prototype]]，在脚本中无法访问到该属性。但在浏览器中可以通过proto访问到。需要明确的是，这个指针连接的是实例与构造函数中的原型对象，而不是实例与构造函数。

虽然我们无法访问到[[Prototype]]，但是可以通过isPrototypeOf()方法来确定对象之间是否存在这种关系。通过Object.getPrototypeOf()来返回[[Prototype]]的值。

当调用实例中的某个属性或方法时，会执行一次搜索，首先从对象实例开始，如果找到了则返回该属性的值。若没有则继续搜索指针指向的原型对象。这个过程有点类似于作用域链。
虽然我们可以通过对象实例访问到原型对象中的值，但是却不能通过对象实例重写原型对象中的值。如果我们在对象实例中添加一个与原型对象中同名的属性，这样会屏蔽原型对象中的那个属性。

auto

cover

contain

此值刚好与cover相反，主要用于将图片缩小。这种方法也会导致图片失真。可能出现水平方向撑满容器，但高度方向不撑满容器。

canvas

ctx.save()

该方法是Canvas 2D API通过将当前状态放入栈中，保存canvas全部状态的方法

ctx.restore();

该方法是Canvas 2D API通过在绘图状态栈中弹出顶端的状态，将canvas恢复到最近的保存状态的方法，。如果没有保存状态，此方法不做任何改变。

在使用上述方法时，要注意的是每当restore之后，栈中保存的状态就被弹出。若save之前再restore，则栈中无保存状态，不弹出。

translate

该方法会改变canvas中原点的位置，主要用于rotate。
注意在rotate后将坐标原点更改回初始位置。

rotate

该方法在变换矩阵中增加旋转的方法。角度变量表示一个顺时针旋转角度并且用弧度表示。
旋转中心点一直是canvas的起始点。如果想改变中心点，我们可以通过translate()方法移动canvas。
