    二、Hybrid/Native/Web App

    参考文章：http://www.uisdc.com/web-hybrid-native-app
  • Web App
    Web App指采用Html5语言写出的App，不需要下载安装。生存在浏览器的应用，基本上可以说是触屏版的网页应用。

    优点

  1. 跨平台，跨终端
  2. 开发成本低，更新快

    缺点

  3. 临时性入口，留存率低
  4. 无法获取系统级别的通知，提醒，动效等
  5. 设计受限多，体验较差
  • Native App
    Native App指的是原生程序，一般依托于操作系统，有很强的交互，是一个完整的App，可拓展性强。需要用户下载安装使用。
  1. 性能稳定，操作速度快，上手流畅，完美的用户体验
  2. 访问本地资源，拥有系统级别的贴心通知或提醒
  3. 用户留存率高
  • Hybrid App
    Hybrid App指的是半原生半Web的混合类App。需要下载安装，看上去类似Native App，但只有很少的UI Web View，访问的内容是Web
    Hybrid App极力去打造类似于Native App的体验，但仍受限于技术，网速等很多因素，尚不完美。

Web受限因素及设计要点

因此需要注意：

  1. 简化不重要的动画／动效
  2. 简化复杂的图形文字，样式
  3. 减少页面渲染的频率和次数
  4. 少用手势，避免与浏览器手势冲突
  5. 少用弹窗
  6. 减少页面内容，控件数量
  7. 减少页面跳转次数，尽量在当前页面显示
  8. 增强Loading时的趣味性
  9. 增强页面的主次关系
  10. 增强控件的复用性

四、MVC/MVP/MVVM

参考文章：http://www.ruanyifeng.com/blog/2015/02/mvcmvp_mvvm.html

MVC

  • 视图(View)：用户界面
  • 控制器(Controller)：业务逻辑
  • 模型(Model)：数据保存
  1. View传送指令到Controller
  2. Controller完成业务逻辑后，要求Model改变状态
  3. Model将新的数据发送到View，用户得到反馈
    所有通信都是单向的。

    MVP

    MVP 模式将 Controller 改名为 Presenter，同时改变了通信方向。
  4. 各部分之间的通信，都是双向的。
  5. View与Model不发生联系，都通过Presenter传递。
  6. View非常薄，不部署任何业务逻辑，成为“被动视图”（Passive View），即没有任何主动性，而Presenter非常厚，所有业务逻辑都部署在那里。

    MVVM

    MVVM模式将Presenter改名为ViewModel，基本上与MVP模式完全一致。

    唯一的区别是，它采用双向绑定，View的变动，自动反映在ViewModel，反之亦然。Angular和Ember都采用这种模式。

三、Flex布局

http://www.ruanyifeng.com/blog/2015/07/flex-grammar.html?utm_source=tuicool

Webkit内核的浏览器例如(Safari和Chrome)，必须加上-webkit前缀。
设置为Flex布局后，子元素的float,clear,vertical-align将失效

采用Flex布局的元素，成为Flex容器。它的所有子元素自动成为容器成员，称为Flex项目。

容器默认存在两根轴：水平的主轴(main axis)和垂直的交叉轴(cross axis)。主轴的开始位置叫做main start，结束位置叫main end。交叉轴的开始位置叫cross start,结束位置叫cross end。

项目默认沿主轴排列。单个项目占据的主轴空间叫做main size,占据的交叉轴空间叫做cross size。

JavaScript的事件机制

一、事件冒泡

IE的事件流叫做事件冒泡，即事件开始时又最具体的元素接受，然后逐级向上传播到较为不具体的节点。

二、事件捕获

事件捕获的思想是，不大具体的节点是比较早接受到信息的，而最具体的节点则最后接受到信息。

三、DOM标准的事件模型

“DOM2级事件”规定的事件流包括三个阶段：1.事件捕获阶段、2.处于目标阶段、3.事件冒泡阶段。首先发生的是事件捕获，然后是实际的目标接受到事件。然后是事件冒泡阶段，对事件作出响应。

兼容浏览器的处理方法

“DOM2级事件”定义的对事件处理的两种方法

  • element.addEventListner(“event”,fn,boolen);
  • element.removeEventListener(“event”,fn,boolen);
    element,event,fn必须相同

ECMAScript

ECMAScript规定了JavaScript脚本的核心语法，如 数据类型、关键字、保留字、运算符、对象和语句等，它不属于任何浏览器。

ECMAScript标准定义了JavaScript脚本中最为核心的内容，是Javascript脚本的“骨架”，有了“骨架”，就可以在它上面进行扩展，如DOM和BOM。
